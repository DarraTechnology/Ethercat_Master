# 部署与调试 / 校准指南 (Commissioning & Calibration)

本文是这些案例**从两台电机到跑通**的实际部署流程 + **校准流程**。各案例 README 只讲它自身的操作；安装设备、配置驱动、导出 `config.deni`、校准这类**通用流程**集中写在这里。

> **先说前提**：Darra EtherCAT 主站**没有纯软件模拟器**，要跑必须有 **真网卡 + 真伺服 + GUI 导出的 config.deni**。但可分档测，CNC 不需要传感器，传送带可用"演示:自动来料"先看逻辑（见各 README 与 §6）。

---

## 1. 硬件安装 (接线)

1. **驱动 + 电机**：电机动力线、编码器线接到各台松下 MINAS A6B 驱动；接 24V 控制电源、STO/急停（按 A6 手册）。
2. **EtherCAT 菊花链**（顺序决定从站编号）：
   ```
   PC 专用网卡 → 驱动1 ECAT-IN(X2A) → 驱动1 ECAT-OUT(X2B) → 驱动2 ECAT-IN → 驱动2 OUT → ...
   ```
3. **专用网卡**：用**一块独立网卡**专跑 EtherCAT，别和上网网卡混用。
4. **(传送带案例) 光电传感器**：NPN/PNP 三线 24VDC，输出接驱动 **EXT1**（Touch Probe 锁存源）或任意 **SI**（DI 源）。CNC 不需要传感器。

---

## 2. 安装主站驱动 + 绑定网卡

1. 按 **SDK 安装包 / 官网**（[ethercat.darra.xyz](https://ethercat.darra.xyz/)）安装 **Darra EtherCAT 主站驱动**，并把主站协议**绑定到那块专用网卡**（NDIS 协议绑定）。
2. **关掉该网卡的节能 / RSS / 各种 offload**（Realtek 尤其要关 RSS）——否则非 IP 的实时帧可能收不全，表现为"链路 Up 但扫不到从站"。
3. 所有案例程序**以管理员身份运行**（裸以太网帧需要提权）。

> 判据：网卡绑定成功后，主站 GUI 选该网卡能**扫到从站**；扫不到先查绑定/网卡设置，别先怀疑线缆（EtherCAT R0：帧发出没回来=链路通，是接收侧问题）。

---

## 3. 导入 ESI + 扫描总线

1. 把松下 ESI `panasonic_minas-a6bf_v1_9_1_0_117_0.xml` 放进主站 GUI 的 **ESI 目录**。
2. 打开 GUI → 选专用网卡 → **扫描** → 应看到 N 台 **MINAS A6B**（从站编号 = 菊花链顺序）。

---

## 4. 配置驱动 (PANATERM)

用松下 **PANATERM** 连每台驱动设：

1. **控制方式**：EtherCAT (CoE)，确认支持 **CSP**（周期同步位置）。
2. **电子齿轮**：建议 **1:1**（命令单位 = 编码器增量）；机械换算（脉冲/mm、齿比）放在**案例里**做，驱动侧保持 1:1，校准更直观。
3. **旋转方向 Pr**：先用默认，方向校准时再改（见 §8）。
4. **(传送带) Touch Probe / 锁存输入**：把 **EXT1**（或某个 SI）分配成"外部锁存 / Touch Probe 触发"。
5. **增益**：用 PANATERM **自整定**先把电机跑稳，再上案例。

---

## 5. 导出 config.deni

1. 在 GUI 里**保持松下默认 PDO 分配**：RxPDO `0x1600`（9 字节）/ TxPDO `0x1A00`（23 字节）。
2. **导出 `config.deni`** → 放到案例的 `EtherCATRestore\EtherCATRestore\config.deni`。
3. `config.deni` **不可手工编写**（带 SHA-256 + 逐字节 PDO/FMMU 启动指令，必须对应真实从站）。轴数由实际扫到的从站数决定，界面自动适配。

---

## 6. 上电安全自检

第一次连接务必确认**上电即安全**：未点"使能"前电机不动；连接后目标位置 = 当前实际位置（无跳变）；报警横幅为绿。任何故障 → 组停 + 全轴去使能 + 闭锁，需"报警复位"解锁。

---

## 7. 两台电机从零到跑通 (Quickstart)

### CNC（2 轴 X/Y）
1. **连接** → 轴表自动 `slave0=X、slave1=Y`。
2. **先低速小目标**：把 进给 F、最大加速度 调小；点预设"矩形"（或自己写一小段）。
3. **设原点**（当前位置 = 程序原点）。
4. 填 **脉冲/mm** 估算值（见 §8.2）、设 **软限位 / 每轴最大速度**（轴表里编辑）。
5. **使能** → **运行**，看是否按预期方向/尺寸走。不对就校准（§8）。

### 传送带（2 轴：主+从 或 双从）
1. **连接** → 轴表。
2. 选 **主轴源 = 虚拟**；从轴 **齿比 1.0 / 相位 0**；选 **抓取轴**（可选）、设抓取区。
3. 低速：**全部使能** → **启动同步** → 勾 **"演示:自动来料"**，看物料跟踪 + 飞行抓取（**不接传感器**）。
4. 接光电后，**捕获源**选 Touch Probe / DI，**武装触发** → 真实捕获。

---

## 8. 校准流程

### CNC 校准（顺序很重要）
1. **轴映射 + 方向**：在轴表"角色"列确认 X/Y/Z 对应物理轴；点动/走一小段，看正方向是否符合机床约定。反了就改驱动方向 Pr（或调机械），**不要靠改代码符号**。
2. **脉冲当量（脉冲/mm）—— CNC 最核心校准**：
   ```
   脉冲/mm = 编码器每转脉冲 × 驱动电子齿比 ÷ 每转机械行程(mm)
   ```
   - 松下 A6 23 位编码器 = **8388608 脉冲/转**，电子齿比 1:1。
   - 直连 **L mm 导程**丝杠：`8388608 / L`。例：10mm 导程 → **838860.8**（案例默认 838861，按此填）。
   - 同步带/带轮：每转行程 = 节径×π 或 齿数×齿距；有减速比 i 时 `8388608×i ÷ 行程`。
   - **实测精修**：命令移动 `D_cmd` mm，用卡尺/千分表量实际 `D_act` mm，修正
     `脉冲/mm_新 = 脉冲/mm_旧 × D_cmd ÷ D_act`，迭代 1~2 次到准。
3. **原点 / 回零**：先把轴移到机械零点（手动点动，或用驱动 HM 回零模式 + 限位/Z 相），再点 **"设原点"** 把该处设为程序坐标原点。
4. **软限位**：量出各轴允许行程，在轴表 **"行程下限/上限(mm)"** 填（程序坐标，相对原点）。下=上=0 表示该轴不限。运行前会**整条路径预检**，超限直接拒绝运行；运行中再越限会**钳到限位 + 报警组停**（防撞机）。
5. **每轴最大速度**：在轴表 **"最大速度(mm/s)"** 填（0=不限）。前瞻规划会把它折算到每段方向，保证**任何方向单轴速度都不超限**（斜线/圆弧也不会让某轴超速）。
6. **加减速整定**：最大加速度、jerk 从小往大调，跑到**不报跟随误差、不丢步、机械不抖**为止；S 曲线模式冲击更小。
7. **最大进给 F**：即"进给"输入，是路径合成速度上限；与每轴最大速度共同约束。

> CNC 的"各种最大限制"都在这了：**软限位（位置）+ 每轴最大速度 + 最大加速度 + 最大加加速度(jerk) + 最大进给**，全部可配并在规划/运行时强制。

### 传送带校准
1. **齿比 / 相位**：齿比 1.0 = 与主轴同步；机械变速带按实测比填（移动主轴 X，量从轴带行程，比 = 从/主）；相位让工位/卡爪对齐。
2. **抓取区 + 传感器偏置（关键）**：量 **传感器到抓取头的距离**（折成主轴度数）→ 填"抓取区 起/止(°)"。校准法：放一件物料触发，jog 主轴直到物料到抓取头，读这段主轴行程 = 抓取区中心。
3. **编码器主轴方向**：编码器主轴模式下确认从轴跟随方向与带运动一致。

## 9. 部署到现场清单

- [ ] 专用网卡 + 主站驱动绑定 OK，扫到全部从站
- [ ] PANATERM 各轴：CSP、电子齿轮 1:1、方向、(传送带)锁存输入、自整定
- [ ] config.deni 默认 PDO 导出并放入 EtherCATRestore（三处一致：源/案例/运行目录）
- [ ] 脉冲/mm 实测校准（CNC）/ 齿比相位校准（传送带）
- [ ] 软限位 + 每轴最大速度 填好（CNC）
- [ ] 加减速整定到不报跟随误差/不丢步
- [ ] 上电安全：不使能不动；急停可用；故障组停 + 复位流程验证
