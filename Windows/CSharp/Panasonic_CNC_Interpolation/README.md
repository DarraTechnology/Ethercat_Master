# 松下 A6 CNC 多轴轨迹插补 (CNC Multi-Axis Interpolation)

基于 Darra EtherCAT Master 类库的 **松下 MINAS A6B 伺服** *CNC 多轴轨迹插补* 运动控制案例，WinForms 界面。
多个伺服轴（X/Y/Z）联动，按 **G 代码** 描述的直线（G01）/ 圆弧（G02/G03）轨迹做 **CSP 周期同步位置插补**，
内置 **梯形 / S 曲线速度规划** + **前瞻 Look-Ahead 拐角衔接**，并提供 **2D 路径预览**。

> 设备型号 **松下 MINAS A6B**（Panasonic），PDO 用 ESI 默认映射（来源 ESI：`panasonic_minas-a6bf_v1_9_1_0_117_0.xml`）。
>
> **部署 / 安装驱动 / 配置驱动 / 导出 config.deni / 校准（脉冲当量、软限位、方向、回零）完整流程见上级目录 [COMMISSIONING.md](../../../COMMISSIONING.md)。**

## 一、什么是多轴轨迹插补

CNC（数控）的核心是让多个进给轴 **协调运动**，使刀具中心沿编程轨迹（直线、圆弧）移动。
本例用 **CSP（周期同步位置，Mode=8）**：主站每个 PDO 周期（1ms，DC Sync0）经插补算法算出轨迹上的下一个点，
把该点的 X/Y/Z 坐标换算成各轴目标位置同时下发，从站做精插补。

```
轨迹弧长 s 每周期推进 Δs = v·dt  →  由 s 映射到几何 (直线/圆弧) 得 (x,y,z) mm
                              →  各轴目标脉冲 = 原点[轴] + round(坐标_mm × 脉冲/mm)
```

只用 CSP（不用 PP）：连续轨迹要求 **每个周期都精确给出新目标位置**，这正是 CSP 的用途；
PP（轮廓位置）是点对点模式，不适合连续插补，因此本例运行模式 `0x6060` 恒为 8。

## 二、速度规划（梯形 / S 曲线）

沿合成路径弧长 `s` 规划进给速度 `F`，受 **最大加速度 a** / **最大加加速度 jerk** 约束：

- **梯形（Trapezoid）**：加速度瞬时取 ±a，速度呈梯形（加速段 → 巡航段 → 减速段）。
- **S 曲线（S-curve）**：加速度本身按 **jerk** 斜坡升降，不瞬时跳变，速度曲线平滑成 S 形，
  机械冲击更小。临近段末时按 `v_exit² = v² − 2·a·剩余弧长` 提前减速到出口衔接速度。

---

## 三、前瞻 Look-Ahead（拐角衔接）

G 代码段排成队列，规划阶段计算相邻段在公共顶点的 **衔接速度**，避免每个拐角都减速到 0：

- **拐角衔接速度（GRBL junction deviation）**：由转角夹角 θ + 偏差容差 δ + 加速度 a 算
  `v_junction = sqrt(a·δ·sin(θ/2) / (1 − sin(θ/2)))`，再 clamp 到 ≤ F。
  - 共线（θ≈0）→ 不减速；
  - 直角 / 掉头 → 衔接速度很低 / 为 0。
- **两遍扫描**：
  - **反向（backward）**：保证每段能从入口速度减速到出口限制（`v_exit² ≤ v_entry² + 2·a·len`）；
  - **正向（forward）**：保证每段能从入口加速到出口；
  - **首段入口 = 0，末段出口 = 0**。

## 三·五、最大限制（软限位 / 每轴最大速度 / 加减速）

CNC 该有的限制本案例**全部可配并强制**（在「轴总览」表或「插补参数」里设）：

| 限制 | 在哪设 | 怎么生效 |
|------|--------|---------|
| **软限位**（每轴行程下/上限 mm） | 轴表「行程下限/上限(mm)」（下=上=0 → 该轴不限） | 运行前**整条路径预检**，任一坐标超限 → **拒绝运行**；运行中再越限 → **钳到限位 + 报警组停**（防撞机） |
| **每轴最大速度**（mm/s） | 轴表「最大速度(mm/s)」（0=不限） | 前瞻规划折算到每段方向，**任何方向单轴速度都不超限**（斜线/圆弧也不超速） |
| **最大加速度**（mm/s²） | 插补参数「最大加速度」 | 速度规划的加速度上限 |
| **最大加加速度 jerk**（mm/s³） | 插补参数「jerk」 | S 曲线模式加速度斜坡上限 |
| **最大进给 F**（mm/min） | 插补参数「进给」 | 路径合成速度上限 |

> 软限位是**程序坐标**（相对「设原点」建立的原点）。脉冲当量 / 回零 / 软限位的具体校准步骤见 [COMMISSIONING.md](../../../COMMISSIONING.md) §8。

## 四、PDO 与控制

- **RxPDO 输出 = 9 字节**（`0x1600`）：控制字 `0x6040` / 运行模式 `0x6060` / 目标位置 `0x607A` / Touch Probe 功能 `0x60B8`。
- **TxPDO 输入 = 23 字节**（`0x1A00`）：错误码 `0x603F` / 状态字 `0x6041` / 运行模式显示 `0x6061` / 实际位置 `0x6064` /
  Touch Probe 状态 `0x60B9` / Touch Probe1 位置 `0x60BA` / 跟随误差 `0x60F4` / 数字输入 `0x60FD`。
- 连接时经 CoE 把 PDO 分配 **强制重映成单 PDO**（`0x1C12=0x1600` / `0x1C13=0x1A00`），并做 **PDO 尺寸自检**
  （驱动实测字节数 vs 结构体 `Marshal.SizeOf` = 9 / 23），防止映射错位。
- 每个从站 `ConfigureDC(1000000)` 启用 **Sync0 = 1ms**（CSP 必须 DC）。
- CiA 402 使能握手由 PDO 控制字驱动：`0x06 → 0x07 → 0x0F`（故障时 `0x80` 复位）。未使能 / 握手中 / 故障时目标位置恒等于实际位置（无跳变）。

## 六、G 代码语法

```
G00 X.. Y.. Z..          ; 快移 (本案例按插补速度走, 安全)
G01 X.. Y.. Z.. F..      ; 直线插补; F = 进给 mm/min (模态, 缺省沿用)
G02 X.. Y.. I.. J.. F..  ; 顺时针 (CW) 圆弧, I/J = 圆心相对起点的增量 (XY 平面)
G03 X.. Y.. I.. J.. F..  ; 逆时针 (CCW) 圆弧
; 行内分号后为注释; 坐标缺省 = 保持上一点对应分量 (模态); 单位 mm
```

- 起终点重合的 G02/G03（I/J 给出圆心）= 整圆。
- **模态续行**：省略 G 的续行沿用上一运动 G（标准 G 代码习惯，如 `G01 X10 F600` 后接 `X20`、`X30`）。
- 预设按钮：**矩形 / 圆 / 圆角矩形 / 五角星** 各生成一段示例填入；G 代码框可输入**任意** G00/01/02/03 程序，**不限于预设**。

## 七、config.deni 用法

程序启动时读取：

```
<可执行文件目录>\EtherCATRestore\config.deni
```

若文件不存在会弹窗提示。生成方式：

1. 用 **Darra EtherCAT 主站 GUI** 扫描总线上的松下 MINAS A6B 伺服从站；
2. **保持松下 A6B 默认 PDO 分配**：RxPDO（`0x1C12`）= `0x1600`（9 字节），TxPDO（`0x1C13`）= `0x1A00`（23 字节）；
3. 导出 `config.deni`，放到上面的 `EtherCATRestore` 目录下。

> 本案例是多轴插补，需要至少 **2 个轴**（X/Y 平面），推荐 **3 个轴**（X/Y/Z）。
> 扫描到的从站默认按顺序指定为 `slave0=X、slave1=Y、slave2=Z`，不足的轴会被禁用；可在「轴总览」表「角色」列改。

## 八、操作步骤

1. **连接**：点「连接」，等待状态变绿「已连接 (CSP 插补)」，总览表列出全部轴并自动指定 X/Y/Z。
2. **改角色**（可选）：在总览表「角色」列双击下拉，重新指定哪个从站是 X / Y / Z（同一角色唯一）。
3. **编 G 代码**：在 G 代码框输入，或点预设按钮（矩形 / 圆 / 圆角矩形 / 五角星）。右侧 2D 预览实时显示编程路径。
4. **设参数**：进给 F、最大加速度、jerk、脉冲/mm（示例 838861，**务必按机械校准**）、速度剖面（梯形 / S 曲线）、前瞻拐角偏差 δ；在轴总览表为各轴设 **软限位（行程下限/上限 mm，下=上=0 不限）** 与 **最大速度(mm/s)**。
5. **设原点**：点「设原点」，把当前各轴实际位置记为程序坐标原点（轨迹相对此原点）。
6. **使能**：点「使能」，各已分配轴进入 `OperationEnabled`。
7. **运行**：点「运行」，开始沿轨迹插补；进度区显示当前段 / 总段、完成 %、当前合成进给；预览红点跟随刀位。
8. **暂停 / 停止 / 急停**：「暂停」冻结推进（位置保持）；「停止」结束轨迹（仍使能）；「去使能 (急停)」立刻松轴。
9. **断开**：点「断开」安全停止并释放主站。

## 十、报警 / 诊断

主界面「控制」下方有 **报警状态横幅**（绿=正常 / 橙=警告 / 红=故障）+ **报警明细表**（时间/轴/严重度/类型/消息/状态）+ **[报警复位]** 按钮；出问题的轴在总览表中整行标红。

### 报警类型

- **跟随误差**：`|实际位置 − 本周期下发目标|` 连续超限 → 故障（默认阈值 50000 脉冲，可按轴在 `AxisController.FollowErrLimit` 整定）。
- **掉OP**：轴在插补运行中掉出 `OperationEnabled` → 故障。
- **驱动故障**：状态字 Fault 或 `0x603F` 错误码非 0 → 故障。
- **软限位**：插补坐标超出该轴 [行程下限, 上限] → 钳到限位 + 故障组停（运行前路径预检已拦大多数，此为运行时防御）。
- **AL状态**：从站 AL 状态码非 `NoError` → 故障（带枚举名）。
- **DC失步**：从站 DC 不同步 / 偏差过大 → 警告。
- **掉站**：从站离线（`SlaveOffline` / `IsLost`）或掉出 OP → 故障。
- **链路断**：主/副链路断或 `master.LinkState` 降级 → 故障/警告。
- **主站掉OP**：主站掉出 OP → 故障。
- **紧急码**：Emergency 事件（错误寄存器 bit4/5 = 故障，否则警告）。
- **丢帧**：PDO 连续丢帧 ≥ 5 → 故障，否则警告。
- **WKC短缺**：实测 WKC < 期望 → 报警并由 `IsLost` 逐轴定位。**铁律：不停 OP、不改期望值、不去使能正常轴**。
- **身份不符**：接错型号 → 故障（组停）。

### 反应：组停 + 闭锁

任一 **故障** 级报警 → **组停**：停轨迹 + **全轴去使能** + **闭锁**。
- 多轴插补是耦合运动，一轴停而其它继续跑会撕裂工件 / 刀具，故默认全停。
- 闭锁期 **[运行] / [使能] 被拦**，必须 **[报警复位]**（要求全轴无驱动故障）才解锁。
- **[去使能(急停)] 永远可用**；从站掉线恢复 **不自动重使能**（需人工 [故障复位] + [使能]）。

### 防误报

- 运行 / 设原点 / 刚使能 / 故障复位后有 **~200ms 宽限期** 不判跟随误差（实际正在追目标，必有大偏差 = 正常瞬态）。
- 跟随误差需 **连续 ~50ms** 超限才报（防单帧 DC 抖动）。

## 十一、线程模型与健壮性

- **PDO 控制回调 `ProcessDataCyclicSync`（约 1ms）**：随总线周期触发，推进轨迹弧长 + 逐轴算目标位置下发 + 写状态快照 + 置报警 `volatile` 闩。
  **严禁 `while + Thread.Sleep` 自旋控制。**
- **StatusUpdateLoop（约 50ms 后台线程）**：经 `BeginInvoke` 刷新总览表 / 进度 / 预览 / 报警，`try/catch` 兜住 `ObjectDisposedException` / `InvalidOperationException`。
- 规划（解析 + 前瞻 + 速度规划）在 UI 线程「运行」时一次性算好，放 `volatile MotionSegment[]` 供 PDO 线程只读；PDO 线程仅维护执行游标（段索引 / 段内弧长 / 当前速度）。
- UI 写控制变量、PDO 写快照，字段用 `volatile` 隔离，互不阻塞。
- 通讯异常不抛未处理异常；断开 / 关闭走统一清理（取消订阅 `ProcessDataCyclicSync` → `Join` UI 线程 → `Abort()` → `Close()`），防 native UAF。

## 十四、自己修改 / 更换硬件

| 想做什么 | 怎么改 |
|---------|--------|
| **换成你的真实从站** | 用主站 GUI 扫描你的真从站导出 `config.deni` 覆盖 `EtherCATRestore/config.deni`。轴数自适应（`master.SlaveCount`），界面自动按扫到的轴数生成。 |
| **换不同型号伺服**（仍 CiA402/CSP） | 若该型号默认 PDO 与松下 9/23 不同：改 `MainForm.cs` 顶部 `PA_Output` / `PA_Input` 结构体，字段顺序按你从站的 PDO 条目逐字节对齐；或保持连接时的单 PDO 重映（`0x1C12=0x1600` / `0x1C13=0x1A00`）若你的从站也支持。连接有 **PDO 尺寸自检**，不符会报错告诉你实际字节数。 |
| **换步进**（如 STF-EC） | 参考同库 `STF-EC_CSP_PP` 例子的 PDO（29/35）与重映，把结构体 / 重映换成步进的即可。 |
| **改 X/Y/Z 对应哪个物理轴** | 运行时在「轴总览」表「角色」列下拉改，无需改代码。 |
| **改脉冲当量（脉冲/mm）** | 界面「脉冲/mm」填机械实测值（校准见 [COMMISSIONING.md](../../../COMMISSIONING.md) §8.2）。 |
| **改软限位 / 最大速度 / 加减速** | 界面直接改（轴表软限位 / 最大速度 + 参数区加速度 / jerk / 进给）。 |
| **改 DC 周期** | 源码 `ConfigureDC(1000000)` 改纳秒数（默认 1ms）。 |
| **改控制逻辑** | 实时控制全在 `OnPdoCycle`（随总线周期触发）；规划在 `PlanLookAhead` / `AdvanceSpeed` / `ParseGcode`。改完用 VS / MSBuild 重编。 |

> 改 PDO 结构体后务必让字段总字节 = 驱动实际 PDO 字节（连接日志会打印实测 输出/输入 字节数）。
