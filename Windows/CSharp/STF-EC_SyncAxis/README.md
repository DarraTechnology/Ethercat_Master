# STF-EC 同步轴 / 电子齿轮 (Synchronized Axis / Electronic Gear)

基于 Darra EtherCAT Master 类库的 **STF-EC 步进驱动器** *同步轴* 运动控制案例，WinForms 界面。
多个轴按各自 **齿比** 跟随一个 **虚拟主轴** 运动 —— 齿比 1:1 即所有轴**同一数据点同时映射**（完全一致运动），
齿比 ≠ 1 即 **电子齿轮**（按固定比例联动）。

> 设备型号 **STF-EC** ×5（VendorId `0x00000168`，Shanghai AMP&MOONS' Automation），与 `STF-EC_CSP_PP` 案例同款硬件、同一 `config.deni`。

---

## 一、什么是同步轴 / 电子齿轮

工业运动控制里，**电子凸轮 (Electronic Cam)** 用一张主从位置曲线让从轴跟随主轴运动。
**电子齿轮 (Electronic Gear)** 是凸轮的**线性特例**：从轴位置 = 主轴位置 × 固定齿比，曲线退化成一条直线。

本例就是电子齿轮：

```
从轴目标位置 = 启动同步时的基准位置(Base) + 虚拟主轴位置(masterPos) × 齿比(Ratio)
```

- **齿比 = 1.000**：所有轴的目标增量完全相同 → 用户口中「同一数据点同时映射」，多个轴像被同一根轴带动一样齐步走。
- **齿比 = 2.000**：该轴走主轴的 2 倍 → 主轴转 1 圈，从轴转 2 圈。
- **齿比 = 0.500**：该轴走主轴的一半。
- **齿比 = -1.000**：该轴反方向等速跟随。

把每个轴齿比设成不同值，就能直观看到比例联动。

---

## 二、为什么只用 CSP（不用 PP）

同步联动要求**每个 PDO 周期都精确给出新的目标位置**，这正是 **CSP（周期同步位置, Mode=8）** 的用途：
主站每 1ms 经 DC（Sync0）把插值后的目标位置下发，从站做精插补。

PP（轮廓位置）是「发一个目标点、驱动器内部走梯形曲线」的点对点模式，**不适合连续同步**，因此本例**完全去掉 PP 分支**，运行模式 `0x6060` 恒为 8。

---

## 三、虚拟主轴

本例不需要一个真实的「主轴」硬件，而是用一个**软件计数器**当虚拟主轴：

- **自动推进**：点「启动同步」后，每个 PDO 周期（约 1ms）主轴位置 `+= 速度 ÷ 1000`。例如速度 = 10000 脉冲/秒 → 每周期 +10 脉冲。
- **手动点动**：按住「◀ 主轴-」/「主轴+ ▶」直接推/退主轴位置，松开即停，方便手动对位观察比例运动。
- **启动同步瞬间**会重新快照各轴当前实际位置作为 `Base`，并把主轴清零，保证从「当前位置」平滑开始，不会跳变。

---

## 四、PDO 与控制（与 STF-EC_CSP_PP 一致）

- **RxPDO 输出 = 29 字节**（`0x1600~0x1603`）：控制字 / 运行模式 / 目标位置 / 轮廓速度等。
- **TxPDO 输入 = 35 字节**（`0x1A00~0x1A03`）：错误码 / 状态字 / 实际位置 / 实际速度 / Touch Probe 等。
- 连接时经 CoE 把 PDO 分配 **重映成完整 4+4**（`0x1C12`/`0x1C13`），并做 **PDO 尺寸自检**（驱动实测字节数 vs 结构体 `Marshal.SizeOf`），防止映射错位。
- 每个从站 `ConfigureDC(1000000)` 启用 **Sync0 = 1ms**。
- CiA 402 使能握手由 PDO 控制字驱动：`0x06 → 0x07 → 0x0F`（故障时 `0x80` 复位）。未使能 / 握手中时目标位置恒等于实际位置（无跳变）。

## 五、config.deni 用法（同 STF-EC_CSP_PP）

程序启动时读取：

```
<可执行文件目录>\EtherCATRestore\config.deni
```

若文件不存在会弹窗提示。生成方式：

1. 用 **Darra EtherCAT 主站 GUI** 扫描总线上的 STF-EC 从站；
2. 导出网络配置为 `config.deni`；
3. 放到上面的 `EtherCATRestore` 目录下。

> 本案例可直接复用 `STF-EC_CSP_PP` 案例的同一份 `config.deni`（硬件相同）。

## 六、操作步骤

1. **连接**：点「连接」，等待状态变绿「已连接 (CSP 同步轴)」，总览表列出全部轴。
2. **全部使能**：点「全部使能」，各轴驱动状态进入 `OperationEnabled`。
3. **设齿比**（可选）：在总览表「齿比」列双击某行直接输入，例如轴 2 填 `2.000`、轴 3 填 `0.500`。默认全为 `1.000`。
4. **启动同步**：设好「速度(脉冲/秒)」后点「启动同步」，主轴位置开始推进，各轴按齿比一起动起来。
   - 齿比都 = 1 → 所有轴目标位置完全一致（同一数据点同时映射）。
   - 齿比不同 → 各轴位置按比例拉开，肉眼可见电子齿轮效果。
5. **手动对位**（可选）：用「◀ 主轴-」/「主轴+ ▶」按住点动主轴，观察各轴比例跟随。
6. **停止 / 急停**：「停止」停止主轴推进（各轴保持当前位置）；「全部去使能 (急停)」立刻松轴。
7. **断开**：点「断开」安全停止并释放主站。

## 八、线程模型与健壮性

- **PdoControlLoop**（约 1ms 后台线程）：推进虚拟主轴 + 逐轴算目标位置下发 + 写状态快照。
- **StatusUpdateLoop**（约 50ms 后台线程）：经 `Invoke` 刷新总览表与主轴位置，`try/catch` 兜住 `ObjectDisposedException` / `InvalidOperationException`。
- UI 写控制变量、PDO 写快照，字段用 `volatile` 隔离，互不阻塞。
- 通讯异常不抛未处理异常，断开 / 关闭走统一清理（取消订阅 → `Abort()` → `Close()`）。

## 报警 / 诊断 (同步轴专用)

主界面「轴总览」下方新增 **报警状态横幅**(绿=正常 / 橙=警告 / 红=故障)+ **报警明细表**(时间/轴/严重度/类型/消息/状态)+ **[报警复位]** 按钮; 出问题的轴在总览表中整行标红。

### 报警分两大类

**同步类**
- **跟随误差**: |实际位置 − 本周期下发目标| 连续超限(默认 5000 脉冲 = 0.5 圈)→ 故障。
- **同步中掉OP**: 轴在同步运行中掉出 OperationEnabled → 故障。
- **驱动故障**: 状态字 Fault 或 0x603F 错误码非 0 → 故障。
- **AL状态**: 从站 AL 状态码非 NoError(如 SyncManagerWatchdog)→ 故障(带枚举名)。
- **DC失步**: 从站 DC 不同步 / 偏差过大 → 警告(DCSyncLost 事件偏差超半周期升故障)。

**通讯类**
- **掉站 / 掉OP**: 从站离线(SlaveOffline / IsLost)或掉出 OP → 故障。
- **链路断**: 主/副链路断 或 master.LinkState 降级 → 故障/警告(端口级精确定位)。
- **主站掉OP**: 主站掉出 OP → 故障。
- **紧急码**: Emergency 事件(错误寄存器 bit4/5 = 故障, 否则警告)。
- **丢帧**: PDO 连续丢帧 ≥ 5 → 故障, 否则警告。
- **WKC短缺**: 实测 WKC < 期望 → 报警并由 IsLost 逐轴定位。**铁律: 不停 OP、不改期望值、不去使能正常轴**(符合 EtherCAT 可观测 + 不停机)。
- **身份不符**: 接错型号 → 故障(组停)。

### 反应: 组停 + 闭锁

任一 **故障** 级报警 → **组停**: 停虚拟主轴 + **全组去使能** + **闭锁**。
- 同步是耦合运动: 一轴停而其它继续跑会撕裂机构, 故默认全停。
- 闭锁期 **[启动同步] / [全部使能] 被拦**, 必须 **[报警复位]**(要求全轴无驱动故障)才解锁。
- **[全部去使能(急停)] 永远可用**。
- 从站掉线恢复 **不自动重使能**: 上线仅清离线报警, 需人工 [全部故障复位] + [全部使能]。

### 防误报

- 启动同步 / 改齿比 / 刚使能 / 故障复位后有 **~200ms 宽限期** 不判跟随误差(实际正在追目标, 必有大偏差 = 正常瞬态)。
- 跟随误差需 **连续 ~50ms** 超限才报(防单帧 DC 抖动)。
- 阈值默认 5000 脉冲(可按轴在 `AxisController.FollowErrLimit` 整定; 产品建议收紧到 500~1000)。

### 线程模型 (报警安全)

- **PDO 热路径(1ms)**: 只算跟随误差 + 置 `volatile` 闩, 完全无锁、不碰 UI/SDK 托管属性。
- **后台(50ms)**: 据闩 + 轮询 SDK 属性(IsLost/State/ErrorCode/链路/DC/WKC)在 `AlarmManager`(内部 lock)升降报警, 再 Invoke 刷 UI。
- **事件回调线程**: 走 `AlarmManager.Enqueue` 无锁队列投递, 由后台 Drain 落地。

> 诊断提示(EtherCAT R0): WKC=0 / 扫不到从站 ≠ 链路断。先看链路断/掉站报警定位真实断点, 别先归因线缆。
